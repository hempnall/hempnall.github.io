--- layout: post title: Qt - unit testing with files date: '2014-05-11T02:47:00.000-07:00' author: James Hook tags: modified_time: '2014-05-11T02:47:10.197-07:00' blogger_id: tag:blogger.com,1999:blog-5482187274444893467.post-327506571744471243 blogger_orig_url: http://computer-says-yes.blogspot.com/2014/05/qt-unit-testing-with-files_11.html --- This is just a quickie blog post building up to creating unit tests for Qt components that have to deal with files on the file system.

I want to be able to store these files as part of the GIT repository of the test project. I accept that the tests will only work if the testing project has to be on the machine when the test suite is run.
In your .pro file you need to define the pre-processor variable PROJDIR 

DEFINES += PROJDIR=\\\"$${_PRO_FILE_PWD_}\\\"

Then in your source code you can access the pre-processor symbol 

#include <QCoreApplication>

#ifndef PROJDIR
#define PROJDIR QString("not defined")
#endif

#include <QDebug>

int main(int argc, char *argv[])
{
    QCoreApplication a(argc, argv);
    qDebug() << PROJDIR ;
    return a.exec();
}

now you can reference test files that are relative to the .PRO file!!!
